🎯Tổng Quan Dự Án

Ứng dụng Food App là một ứng dụng công thức nấu ăn di động được xây dựng bằng React Native và Expo. Ứng dụng này đã được tích hợp tính năng Semantic Search (Tìm kiếm Thông minh) để giúp người dùng tìm công thức nấu ăn một cách thông minh, hiệu quả hơn.

Mục tiêu chính:

  • Cải thiện trải nghiệm tìm kiếm người dùng
  • Hỗ trợ tìm kiếm bằng ngôn ngữ tự nhiên
  • Tăng độ chính xác kết quả tìm kiếm
  • Cung cấp gợi ý thông minh theo ngữ cảnh

✨Tính Năng AI Được Thêm

1. Semantic Search (Tìm Kiếm Thông Minh)

Người dùng có thể tìm kiếm bằng mô tả tự nhiên thay vì chỉ từ khóa chính xác.

Ví dụ: Thay vì tìm "Mì Ý", người dùng có thể tìm "cái gì nhanh và lành mạnh"
Hệ thống sẽ hiểu được ý nghĩa và trả về công thức phù hợp.

2. Hybrid Search (Tìm Kiếm Kết Hợp)

Kết hợp tìm kiếm theo ý nghĩa (semantic) với lọc theo danh mục (category).

  • Người dùng nhập truy vấn tự nhiên
  • Người dùng có thể chọn chip danh mục để lọc
  • Kết quả được sắp xếp theo độ liên quan

3. Smart Suggestions (Gợi Ý Thông Minh)

Gợi ý động khi người dùng nhập tìm kiếm.

  • Tên công thức phù hợp
  • Đặc tính (nhanh, lành mạnh, cay, v.v.)
  • Tối đa 5 gợi ý để không gây bối rối

⚙️Kỹ Thuật Chi Tiết

semanticSearch.ts - Các Hàm Chính

1. cosineSimilarity()

function cosineSimilarity(vecA: number[], vecB: number[]): number

Tính độ tương đồng giữa hai vector sử dụng công thức cosine similarity.

  • Đầu vào: 2 vector số học
  • Đầu ra: Giá trị từ 0 (hoàn toàn khác) đến 1 (giống hệt)

2. generateEmbedding()

function generateEmbedding(text: string): number[]

Chuyển đổi văn bản thành vector số (embedding).

  • Dùng hash-based approach
  • Trả về embedding 50 chiều
  • Đã chuẩn hóa (normalized)

3. semanticSearch()

function semanticSearch(query, recipes, limit)

Tìm kiếm công thức dựa trên ý nghĩa của query.

  • Tạo embedding cho query
  • So sánh với embedding của các công thức
  • Tăng điểm nếu có keyword match
  • Sắp xếp theo độ liên quan giảm dần

4. hybridSearch()

function hybridSearch(query, recipes, selectedCategories, limit)

Kết hợp semantic search với category filter.

  • Gọi semanticSearch với kích thước lớn hơn
  • Lọc kết quả theo danh mục được chọn
  • Trả về top N kết quả

5. getSearchSuggestions()

function getSearchSuggestions(query, recipes)

Trả về danh sách gợi ý cho người dùng.

  • Gợi ý recipe titles khớp query
  • Gợi ý characteristics (nhanh, lành mạnh, v.v.)
  • Giới hạn 5 gợi ý

🔧Cấu Hình & Tùy Chỉnh

1. Điều Chỉnh Embedding Dimension

Mở semanticSearch.ts, dòng 47:

const embedding = new Array(50).fill(0);  // Thay 50 thành số chiều khác
  • Chiều cao hơn: Chính xác hơn (nhưng chậm hơn)
  • Chiều thấp hơn: Nhanh hơn (nhưng chính xác thấp hơn)
  • Khuyến nghị: 30-100

2. Điều Chỉnh Keyword Boost

Mở semanticSearch.ts, dòng 79:

keywordBoost += 0.15;  // Thay 0.15 thành giá trị khác
  • 0.05: Keyword ít ảnh hưởng
  • 0.15: Mức trung bình (hiện tại)
  • 0.30: Keyword rất ảnh hưởng

3. Điều Chỉnh Ngưỡng Min Score

Mở semanticSearch.ts, dòng 83:

if (finalScore > 0.2) {  // Thay 0.2 thành giá trị khác
  • 0.1: Kết quả nhiều (có thể không liên quan)
  • 0.2: Mức trung bình (hiện tại)
  • 0.5: Kết quả ít nhưng rất chính xác

4. Thêm Characteristics Mới

Mở semanticSearch.ts, dòng 64:

const characteristicKeywords: Record<string, string[]> = {

📊Hiệu Suất

Độ Phức Tạp Thuật Toán

  • Time Complexity: O(n × d)
    • n = số công thức
    • d = embedding dimension (50)
  • Space Complexity: O(n × d)

Bảng So Sánh Hiệu Suất

Số Công Thức Thời Gian Tìm Kiếm Khuyến Nghị
< 100 < 10ms OK, không cần tối ưu
100-1000 10-50ms Xem xét caching
> 1000 > 100ms Dùng API bên ngoài
✅ Tình Trạng Hiện Tại: ~50 công thức → ~ 5ms (Rất nhanh)

🔌Tích Hợp API Bên Ngoài (Nâng Cao)

Nâng Cấp lên OpenAI API

Để cải thiện độ chính xác, bạn có thể thay thế embedding local bằng OpenAI API:

async function generateEmbeddingWithOpenAI(text: string): Promise<number[]> {
  const response = await fetch('https://api.openai.com/v1/embeddings', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${OPENAI_API_KEY}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      input: text,
      model: 'text-embedding-3-small',
    }),
  });

  const data = await response.json();
  return data.data[0].embedding;
}

Lợi Ích & Nhược Điểm

Lợi Ích Nhược Điểm
✅ Chính xác cao hơn 30-40% ❌ Yêu cầu API key
✅ Hiểu ngôn ngữ tự nhiên tốt hơn ❌ Có chi phí
✅ Hỗ trợ đa ngôn ngữ ❌ Yêu cầu kết nối internet

Giá Tiền OpenAI

Text-embedding-3-small: $0.02 per 1M tokens

Với ~50 công thức × 20 tokens/công thức = 1000 tokens ≈ $0.00002 per search
